## Architecture decisions
- **PDF**: jsPDF + jspdf-autotable chosen over @react-pdf/renderer (known RSC conflicts in Next 16)
- **Map component**: @react-google-maps/api with controlled polygons, satellite/hybrid view, Places Autocomplete restricted to Canada
- **Manual vs Map**: Toggle button switches between map drawing and manual sqft entry. Manual fallback always available even in map mode (for phone quotes). E2E tests use manual mode for CI reliability
- **Pricing**: Server-authoritative. Client sends surfaces, server recalculates from catalog. Never trusts client-provided prices
- **Quote -> Job**: Direct insert into jobs table (avoids circular dependency with createJobAction)
