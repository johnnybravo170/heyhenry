# HeyHenry UX Redesign вЂ” Pipeline & Screen Status

**This is the single source of truth for the redesign pipeline. Every design/build session reads it FIRST and updates it on handoff.** It exists to kill the #1 time sink: re-discovering what's already done.

## The pipeline (three sessions, three lanes)

```
RESEARCH в”Ђв”Ђв”Ђbriefsв”Ђв”Ђв–¶ OD DRIVER в”Ђв”Ђв”Ђrenders+cardsв”Ђв”Ђв–¶ CODING
(analysis)            (the bottleneck)               (the Mini)
```

| Session | Owns (writes) | Produces | Reads |
|---|---|---|---|
| **Research** | `docs/ux/briefs/*.md` + this tracker | grounded screen briefs (workflow analysis, current-vs-target, real-code grounding) | vault Module docs, real code, foundation docs |
| **OD Driver** | OD prompts, `od-*/` renders, **Dev cards** on the Ops `dev` board | hi-fi renders + critique + the build cards | the brief for the screen |
| **Coding** (Mini) | `src/**` + commits | the implemented screen | the brief + the render + the dev card |

**Lane discipline (this prevents the duplicate-work we hit):** one session owns each artifact type. Research writes **only briefs** (+ this tracker). OD Driver owns **prompts, renders, and dev cards**. Coding owns **commits to `src`**. Don't create dev cards from Research; don't write briefs from OD Driver. *(The duplicate Schedule card happened because card-creation wasn't lane-owned.)*

**Verified = the Coding done-gate (no separate QA session) вЂ” desktop AND mobile.** Before flipping a screen to вњ… Built, run `heyhenry-design-critique` against the **live built screen at BOTH a desktop and a phone viewport** (not the OD render). A screen isn't вњ… until its **mobile** pass clears against the brief's "Mobile vs desktop" section + the rubric's Mobile/field-viability dimension. Log residual designв†”code variance as a Notes entry here + a follow-up card if material. Same critique skill, pointed at the app вЂ” keeps us to three lanes and off the OD-Driver bottleneck.

## Session-start protocol (do this every time, every session)

**0. Work in your OWN git worktree вЂ” never share one checkout.** Multiple agents on a single working copy race on git state вЂ” divergent commits, duplicate-SHA history, stash churn, even transient `rev-parse`/ref failures (we hit all of these). Each session gets its own worktree on its own branch:
```
git worktree add .claude/worktrees/<session> -b ux/<session> origin/main
cd .claude/worktrees/<session>
bash scripts/setup-worktree.sh   # symlinks .env files; idempotent
pnpm install                     # only if node_modules is empty
```
Research / OD Driver / Coding each run in a separate worktree so they never step on each other. See `AGENTS.md` В§"Working in a worktree". *(The Coding/Mini lane already works on PR branches вЂ” this brings the design lanes in line.)*

> **вљ  OD Driver caveat вЂ” the worktree doesn't isolate renders.** The OD daemon writes renders to its project's *fixed* `baseDir`, so a fresh per-session worktree never receives them вЂ” they land in whatever checkout the daemon points at. So the OD Driver lane keeps **one stable checkout = the daemon's `baseDir`** (don't recreate it per session) and stays clear of the other sessions with `git pull --rebase` before every commit/push вЂ” scope commits to `od-*/` renders + this file's cell (Ops `dev` cards aren't git, so no race there). The ephemeral `ux/<session>` worktree is really a **Coding-lane** device вЂ” it isolates `src/` edits, which the OD Driver doesn't make. To isolate the OD Driver fully, point the daemon at a dedicated persistent OD checkout and leave it there.

1. **`git pull --rebase origin main`** вЂ” never work stale. (Most cross-session mixups this arc came from a stale checkout.)
2. **Read this file** вЂ” find your screen's row; confirm the upstream stage is done before you start yours.
3. **Work only your lane's artifact.**
4. **On handoff, update this file** вЂ” flip your stage's cell + drop the ref (commit SHA / card id / render path). Commit it with your work.
5. **Transport is git + the Ops board, not copy-paste.** Briefs live in `docs/ux/briefs/`, renders in `od-*/`, cards on the `dev` board (`epic:ux-redesign`). Read from there; the human is a checkpoint, not the courier. *(OD step is going headless via the `od-redesign-loop` skill.)*

**Claim before you start (two research agents now run in parallel).** Before working a screen, mark its **Brief** cell `рџџЎ (in progress: <your-session>)` and commit that line вЂ” so two researchers never write the same brief. Flip to вњ… when you hand off. Same rule for any lane picking up a screen.
