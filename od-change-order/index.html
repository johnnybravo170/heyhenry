Diff-action palette

One intentional, token-sourced tone per action — always paired with a label + glyph so it’s never colour-only. Used identically in the editor and the public diff view. Separate semantic system from lifecycle status (those use status-tokens.ts draft / pending / approved / declined / voided).
+ADD
ok-soft / ok-ink. Net-additive scope — new line, new work, customer pays more. Background tint on the editor row.
↔CHANGE
info-soft / info-ink. Existing line modified (qty / unit price / both). Semantic-neutral — not "good" or "bad", just changed. Shows a was sub-line for grounding.
−REMOVE
danger-soft / danger-ink. A signed-scope line being struck from the contract. Line-through on the label so the deletion reads from across the room. Delta renders in ok (the project total drops) — tone signals contractual seriousness, not moral judgment.
□BUDGET
chip-fill / muted-ink. Envelope-level shift (modify_envelope) — the coarser, no-line-detail signal. Used only when the category has no line-level diffs (avoids double-counting).

Brief alignment

  • A CO is a mini-estimate — same line/category/markup vocabulary, same price-only customer doc, same typed-name approval. Designed in diff mode against the signed scope, not as a separate app.
  • Diff is legible at a glance — changed rows lead, each as Before → After → Δ. Action chips on the row + a subtle tint on the row background; line-through for removes; was microcopy under changes. Same vocabulary in editor and on the public page.
  • Sticky total delta + operator margin read — the impact bar pins at the top of the editor with the +$X total. An operator-only mono whisper sits next to it: MARGIN ON CHANGE · 22.4% in warn-soft with an OPS tag, so the GC knows whether the CO holds margin. The customer never sees this.
  • Lives inside the Budget tab — breadcrumb is Projects › Hillcrest › Budget › New change order; the project sub-tab nav shows Budget active with a small CO · draft badge. No reintroduced top-level Changes tab (honors decision 6790ef2b + the Hub-locked IA).
  • Plain-English “What’s changing & why” — a peach (rust-soft) card on the public page, above the diff table. This is the trust-moment piece the brief flags as missing today. The editor labels the description field with the hint (plain English — customer reads this) so the operator writes for Helen, not for the audit log.
  • Send preview reuses the estimate send-bar pattern — rendered customer doc on the left, send controls on the right (recipients checklist, optional personal note, copyable approval link, info-soft Email + SMS · both go out channel chip). On mobile it's a bottom sheet with the same elements stacked.
  • Lifecycle states — Draft (the editor) · Pending (sent, awaiting customer, realtime) · Approved (applied banner + a peach ✪ HENRY · Now billable · roll into draw 3? nudge — this is the approved-but-unbilled leak the brief calls out) · Declined (with reason + Clone-to-draft) · Empty (calm).
  • Rust stays the single accent — primary CTA + the ✪ HENRY chrome only. Henry fill reflects meaning: peach for draft-ready / ready-to-bill (never danger-red on a positive). The OPS margin whisper uses warn-soft, not rust — the rust budget is preserved.
  • Three type sizes on every surface (16 / 14 / 12) plus mono labels (11) as a labelling device only — eyebrows, column headers, currency hashes, IDs. Money is tabular-nums, right-aligned, de-emph cents everywhere, including the impact bar and the public fee breakdown.
  • Public page = price-only — cost / markup never appears. Fee breakdown shows Cost of work · Management fee (12.00%) · Total. New-vs-was project total. was $164,000.00 reads as a strikethrough; the new total reads in ink.
  • Approve = typed-name e-signature; decline takes an optional reason; the 3-state form (pending → approve / decline → done) is sketched as mini-state previews under the live form. The desktop frame surfaces all three sub-states so the build team can see the whole flow at once.
  • WCAG · never colour-only — every action chip pairs the colour with a mono label and a glyph; line-through carries the removal signal independently of the danger tint.
