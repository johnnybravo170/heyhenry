Brief alignment

  • Lean identity-only header — name + ▾ chevron + Active badge on line 1; muted linked customer on line 2; ghost “✦ Add” + “⋯” overflow on the right. No metrics in the persistent chrome — carries across every tab unchanged (Overview & Budget shown).
  • Project Details popover — consolidates name, customer, description, dates, billing, status into one card + a Crew checklist with each worker’s default pay/charge rate read-only and a per-row “▾ override” disclosure.
  • ONE unified nav — 9 tabs (Budget · Spend · Labour · Schedule · Billing · Overview · Client · Photos · Documents). Soft warn-tone “²” count on the Budget tab (the alerts’ owning surface); rust “²” on Client for unread messages. Mobile collapses to a select with a total badge on the trigger + per-tab badges in the dropdown.
  • Alert surfacing model — same alerts surface in three layers: as chips on their owning tab (Budget shows margin-at-risk + unsent scope changes), as a count badge on the tab label (Budget²), and as a row in the Overview “Needs you” aggregator. No stacked banners; no duplicate homes.
  • Henry-fill = meaning — danger-soft for at-risk, rust-soft peach for ready-to-bill (positive, never danger-red), warn-soft for unsent changes, info-soft for client message + CO chips, ok-soft for accepted sub-quotes. Sparkles reserved for genuine Henry insights (the Budget-card diagnostic strip on Overview, the “✦ Add” capture button) — not on every alert row.
  • Budget — execution posture — the same component as the estimate’s authoring face, actuals-forward. Columns Category · Estimate · Spent · Committed · Remaining (+ thin multi-segment bar). Sections default collapsed; one expanded (Trades). One category drill-in (Plumbing) with cost lines + “Spent by source” deep-links. Calm signed-estimate banner anchors the top above the summary.
  • Dual budget read — “53% complete · 61% of budget consumed · $86,100 of $142,000.” Two distinct metrics; the gap is the margin story. Never merged.
  • Three type sizes — 28 title, 16 panel titles, 14 body, 12 secondary; 11px mono uppercase for eyebrows/labels. Tabular numerals on every money figure; cents demoted.
  • Single accent — rust is reserved for the “Bill draw” CTA, the ✦ Add button, and Henry insight glyphs only. Status uses soft-pair tones (label + glyph + small chip), never raw red/amber/blue blocks.
